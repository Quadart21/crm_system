## Rick CRM v1.6.0

Первый публичный релиз **Rick CRM** — система для управления партиями (МК), курьерами, статистикой, финансами, тикетами/диспутами и выплатами.

### Технологии

- **Фронтенд:** React 19, Vite 7, Tailwind CSS 4, shadcn/ui, TanStack Query
- **Бэкенд:** Hono, typed-rpc, Prisma ORM, SQLite
- **Платформа:** Adaptive AI SDK

### Что входит в релиз

- Ролевая модель: супер-админ, админ, саппорт, курьер
- **Партии (МК):** создание, выдача курьеру, закрытие, цена товара, ФАС, оплата склада, себестоимость 1 г
- **Ввод данных:** адреса, распределение по маркетплейсам, расчёт заработка
- **Финансы:** распределение на руки / залог / банк, выплата банка с бонусами и финальными списаниями по диспутам
- **Тикеты и диспуты:** номер заказа, автоподбор ввода, расчёт списания, settlement-история
- **Дашборд:** аналитика по периоду, МП, курьерам, финансам и диспутам
- **Настройки:** города, товары, МП, статусы курьера, типы проблем, решения диспутов, бонусы
- **Логи** действий сотрудников

### Запуск локально (Windows / Linux)

```bash
# Перед установкой (для пакета @adaptive-ai/sdk)
export ADAPTIVE_AI_REGISTRY=https://registry.npmjs.org/

npm install
npx dotenv -e .env.development -- prisma migrate deploy

# API (порт = PORT + 1, по умолчанию 5191)
npx dotenv -e .env.development -- npx tsx src/api/server.ts

# Фронтенд (порт = PORT, по умолчанию 5190)
npx dotenv -e .env.development -- npx vite --host 0.0.0.0
```

Откройте в браузере: `http://localhost:5190/` (порт задаётся в `.env.development`).

### Известные ограничения

- Пароли сотрудников хранятся в открытом виде (MVP) — перед боевой эксплуатацией нужно хэширование
- Авторизация через `actorLogin` без полноценных сессий
- Часть требований из ТЗ (выгрузки курьер/админ, план МК, франшиза) — в следующих версиях
